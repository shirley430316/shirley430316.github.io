User Portal Backend Code

1. Overview

Spring Boot layered architecture is a well-organized pattern. It mainly contains database layer, persistence layer, business layer and presentation layer.

Database layer contains all the databases, in this case, MySql.

Persistence layer contains the logic of database storage, which is implemented by DAO (data access object) and Repository class in my case. It links transferred object and database.

Business layer contains the business logic, consisting of Service classes.

Presentation layer is the top layer of the architecture and handle the frontend HTTP requests by Controller classes.

2. DAO

We start with @Entity annotation to indicate that, at an abstract level, the class is correlated with a table in the database.

Serializable is a marker interface which has no methods. It simply indicates that User class could be serialized to a byte stream and then stored in database.

When running the application, Spring Boot will automatically generate MySql query and create the table in database.

3. Repository

Define a UserRepository class to manipulate data. The methods can be used in business layer.

4. DTO

DTO (data transfer object) could be similar to DAO, but it is not necessarily corelated to the database table. It is normally used between presentation layer and business layer. Compared to use several parameters, DTO is safer (for example, it wouldn’t expose infomation in URL) and more modulized.

4.1 UserDTO

UserDTO is designed to transfer data when the user registers. In practice, it should store unencrypted password, while User the DAO class stores the encrypted one. But I don’t wanna write a encrypt class now, so whatever.

4.2 UserLoginDTO

UserLoginDTO is used when the user login by password.
