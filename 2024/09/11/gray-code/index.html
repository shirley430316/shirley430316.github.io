On the first lecture of COMP2119 Introduction to Data Structures and Algorithm, Prof. Kao played a game.

He took out 3 coins and flipped them randomly. The result should be 2 cases. If the 3 coins are on the same face, we win. Else, game continues. We can name a coin and flip it. We are not able to see the coins until the game ends. BTW, he is such an amazing professor.

Question: Is there a set of process, which will definitely lead to success?

However, what if coins are more than three?

1.2 Four Coins

To make four coins on the same side, we should make sure the first three coins are on the same side. If they are on the same side, flip the fourth one, and then we win!

However, when we adopt 1->2->1 strategy to three coins, it is possible to successfully make three coins on the same side, after each and every step. That’s why we would better flip the fourth coin every time after we flip another coin.

Simply insert 4 between each step. We get 4->1->4->2->4->1->4. To make it clearer, we could change 1,2 to 3,2, since each coin is equivalent. Then the final strategy is 4->3->4->2->4->3->4.

1.3 More Coins?

The rest of this problem becomes solvable. When it comes to five coins, we firstly make sure the first four coins are the same. However, since we don’t know whether the first four coins are the same during “4->3->4->2->4->3->4”, we assume they are and flip 5, between each step. The strategy becomes 5->4->5->3->5->4->5->2->5->4->5->3->5->4->5.

Similarly, when it comes to six coins, we insert 6 in the “five-coin-strategy”. So on and so forth.

2 Why Gray Code?

We all know that the binary code is widely used in many devices. However, there exists some problems.

For example, suppose we want to change 3 to 4 in a binary system. 3 is represented by 011, and 4 is represented by 100. Though these two numbers are next to each other, they are totally opposite by bit. Thus, due to asynchronicity caused by physical condition, the changing process might be 011->010->110->100, which leads to confusion.

One really smart solution to this error is Gray code, or reflected binary code (RBC). Between two Gray code for successive values, there would be only one bit differing. Just like naming a coin and only flipping it.
